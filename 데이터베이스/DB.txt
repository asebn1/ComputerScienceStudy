
### DBMS
데이터베이스 내 데이터에 접근하도록 도와주는 시스템입니다. 크게 질의처리기와 저장시스템으로 이루어져 있습니다.

### 옵티마이저
가장 효율적인 방법으로 SQL을 수행할 최적의 처리경로를 생성해주는 DBMS의 핵심엔진

### DDL/DML/DCL
DDL(정의어) : DB의 구조를 생성, 수정 삭제
      - create, alter, rename, truncate(테이블 데이터삭제)
DML(조작어) : 데이터의 검색, 삽입, 삭제, 수정
      - select, update, delete
DCL(제어어) : DB의 접근과 사용권한을 주는 언어
      - grant, revoke

### join
1. inner join : 공통적으로 존재하는 데이터 집합. A∩B
2. outer join (left, right, full)
 - left outer join : 공통적으로 존재하는 데이터와, 왼쪽에 명시된 테이블에 존재하는 데이터 집합. 
 - full outer join : 양쪽 테이블 데이터 집합
3. cross join : 모든 조합
4. self join : 같은 테이블 안에서 연결


### JDBC / ODBC
JDBC : 자바와 DB를 연결하기 위한 자바 API입니다. 
ODBC : 마이크로소프트사에서 만들어진 DB에 접근하는 소프트웨어 표준 규격입니다. 윈도우용 프로그램을 DB와 연결시켜줍니다. 

### 인덱스 /
인덱스 알고리즘
추가적인 저장공간을 활용하여 DB 테이블의 검색속도를 향상시키기 위한 자료구조입니다. 대표적인 알고리즘으로 해시 인덱스 알고리즘과 B Tree 인덱스 알고리즘을 사용합니다. 
- 해시테이블은 key, value로 데이터를 저장하여 빠른 데이터 검색이 가능합니다.
- B Tree : 이진트리의 경우 균형이 맞지 않으면 검색 효율이 선형검색 급으로 떨어집니다. 트리의 균형을 자동으로 맞춰주는 로직이 적용된 것이 B Tree입니다.
- 여기에 빠른 접근을 위한 인덱스 노드가 추가로 있는 것이 B+Tree입니다.

### 인덱스 종류
클러스터드 인덱스
넌클러스터드 인덱스
 클러스터드 인덱스는 항상 정렬을 유지하기 때문에 조회성능이 보장됩니다. 하지만 정렬상태를 유지하기 때문에 삽입, 삭제같은 경우 느려집니다.
 넌클러스터드 인덱스는 테이블 정렬이 아닌 별도의 인덱스 페이지를 별도로 가집니다. 때문에 삽입, 삭제 속도가 더 빠릅니다. 남용할 경우 시스템 성능을 떨어뜨릴 수 있습니다.

### 클러스터
디스크로부터 데이터를 읽어오는 시간을 줄이기 위해 자주 사용되는 테이블의 데이터를 디스크의 같은 위치에 저장시키는 방법

### ORACLE
MYSQL
PostgreSQL
 ORACLE은 유료이며 고성능 트랜잭션 처리를 제공하여 속도가 빠르며 대규모 DB를 지원합니다.
 MYSQL은 무료이며 문자열 대소문자를 구분하지 않고 간단한 트랜잭션을 위한 웹 기반 프로젝트에 널리 사용됩니다.
 PostgreSQL은 무료이며 클러스터 기능을 제공하고 대규모 시스템에 널리 사용됩니다.

### 키
후보키 : 유일하게 식별할 수 있는 속성들의 부분집합
기본키 : 후보키 중에서 선택한 주키
대체키 : 기본키를 제외한 나머지 후보키
슈퍼키 : 유일성은 만족하지만, 최소성을 만족하지 못하는 집합
외래키 : 다른 릴레이션의 기본키를 참조

### 무결성
DB에 저장된 값과 실제 값이 일치해야 하는 것을 의미합니다.
* 종류
 - 개체무결성 : 기본키에는 NULL이나 중복 값을 가질 수 없다.
 - 참조무결성 : 외래키에는 NULL이 있으면 안되며 참조가 가능해야함
* 무결성 유지방법
 - 트리거(조건), 제약 조건, 위반 조치

### 시퀀스
기본키와 같이 순차적으로 증가하는 유일한 값을 생성합니다. 

### 뷰
하나 이상의 테이블의 논리적인 부분집합이며 뷰를 통해 테이블 접근 보안 역할을 합니다.

### 트리거
방아쇠라는 뜻으로 방아쇠를 당기면 총기 내부가 알아서 일련의 작업을 실행하는 것을 뜻합니다. 특정 이벤트가 발생하면 자동으로 동작하도록 작성된 프로그램입니다.

### 정규화
DB 설계에서 데이터 중복을 최소화하도록 구조화하는 과정입니다.
- 1 정규형 : 도메인 모두 원자값
- 2 정규형 : 부분 함수 종속 제거
- 3 정규형 : 이행 함수 종속 제거
- BCNF : 결정자 중 후보키가 아닌 것들은 제거
- 4 정규형 : 다치 종속 제거
- 5 정규형 : 조인 종속 제거

### 함수적 종속
릴레이션 속성 X, Y 중, X값이 Y값을 결정한다면,
Y가 X에 함수적으로 종속되었다고 합니다.

### 반정규화
시스템의 성능 향상 및 편의를 위해 정규화된 데이터모델을 통합, 분리하는 과정입니다. 

### 트랜잭션(ACID)
DB의 상태를 변환시키는 하나의 논리적인 작업의 단위입니다. 하나의 트랜잭션은 commit되거나 rollback 됩니다. DB의 무결성과 일관성을 위해서 트랜잭션은 4가지 특징은 만족해야 합니다.
- 원자성은 트랜잭션 작업이 모두 성공하거나 모두 실패해야 하는 것
- 일관성은 일관성있는 DB상태를 유지하는 것
- 독립성은 동시에 실행되는 트랜잭션은 서로에게 영향을 미치지 않음
- 영속성은 트랜잭션 완료시 결과가 영구적으로 반영되어야 함

### 트랜잭션 격리 수준
동시에 DB에 접근할 때 그 접근을 어떻게 제어할지에 대한 설정
- Read Uncommited : 트랜잭션의 commit 이전 조회 허용
 : Dirty Read : B 28 / A(28->27 커밋전) / B 27

- Read Commited : 트랜잭션의 commit이 완료된 것만 조회 허용
 : Non-Repeatable : B 28 / A(28->27 커밋) / B 27

- Repeatable Read : 트랜잭션의 commit이 완료된 것만 조회 허용. 수정 불가
 : Phantom Read : B 27 / A(28 삽입) / B 27 28

- Serializable : 한 트랜잭션 진행 중 다른 트랜잭션에서 수정, 입력 불가
                 격리성 높지만 성능 떨어짐

### 파티셔닝
테이블을 컬럼 단위로 나누는 기법입니다. 장점은 insert나 update 같은 작업이 분산되어 성능이 향상됩니다. 단점은 테이블간 join 비용이 증가합니다.

### 샤딩
테이블을 row 단위로 나누는 기법입니다. 샤드 key를 정하는 방법에 따라서 샤드 종류가 결정됩니다. 크게 Hash 샤딩과 Dynamic 샤딩이 있습니다.
### 클러스터링
DB 서버가 죽는 것을 방지하기 위해 여러 개의 DB를 수평적인 구조로 구축하는 방식입니다.

### 리플리케이션
저장된 데이터가 손실되는 것을 방지하기 위해 Master에서 Slave에 데이터를 복제합니다. 스토리지가 1개여서 발생할 수 있는 손실을 방지합니다.
### NoSQL
Not Only SQL의 약자로 SQL을 보완한다는 의미를 가지고 있습니다. 고정된 스키마가 없어서 데이터를 조회하고 삽입하는 속도가 빠릅니다. 대량의 분산 데이터를 저장하는데 특화되어 있습니다. 

### 분산형 구조 특징: CAP
C(일관성) : 모든 노드가 같은 시간에 같은 데이터
A(가용성) : 노드가 다운되어도 다른 노드에 영향이 없다
P(분산 허용) : 일부 메시지 손실에도 시스템 정상동작
### 데이터 저장방식에
따른 분류
1) Key/Value Database : Redis(레디스), Oracle Coherence
 메모리에 저장하여, 고속 읽기와 쓰기에 최적화되어있습니다. 이를 방지 하기위한 방법으로 snapshot, AOF(쿼리저장, 셧다운시 재실행)
2) Big Table Database (Ordered Key/Value) : Hbase, 카산드라
 보통의 NoSQL은 order by같은 정렬기능을 제공하지 않지만, 이 모델은 내부적으로 key를 정렬이 이루어지며 집계처리 및 속도가 빠릅니다.
3) Document Database : MongoDB, CouchDB, Riak
 key/value store의 확장된 형태로, value에 Document라는 타입을 저장(Document : XML, JSON 등)합니다. 이에 복잡한 데이터 구조 표현 가능합니다.
4) Graph Database : Sones, AllegroGraph, neo4j
 저장방법은 key/value 방식이며 모든 노드는 끊기지 않고 연결되어 있습니다.
