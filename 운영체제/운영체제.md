
### 운영체제란?
시스템의 자원과 동작을 관리하는 소프트웨어, 프로세스, 저장장치 네트워킹, 사용자, 하드웨어 관리

### 실행파일 생성과정
코드 -> 전처리기(치환) -> 컴파일(어셈블리어) -> 어셈블러(기계어) -> 링커 -> 생성

### 명령어 사이클
CPU가 메모리로부터 프로그램 한 개의 기계어 명령어를 가져와 요구 동작을 결정 및 수행하는 과정입니다.
- fetch(인출) : CPU에서 명령어를 읽어오는 단계
- 실행 : CPU가 명령어를 실행하는 단계

### 전역변수 vs 정적변수
공통점 : 둘다 해당 프로그램이 끝날때까지 유지
차이점 : 범위가 다르다. 전역변수는 어느 함수에서든 접근이 가능한 반면, 정적 변수는 변수가 선언된 함수 내에서만 접근 가능하다.

### Byte Ordering이란?
데이터가 저장되는 순서
1. 빅 엔디안(안녕) : 최상위비트가 낮은 주소에 위치하는 저장방식, 네트워크에서 데이터 전송시 사용
2. 리틀 엔디안(녕안) : 최상위비트가 높은 주소에 위치하는 방식, 마이크로프로세서에서 사용

### System call
OS가 애플리케이션에 제공하는 interface
1. folk() : child 프로세스를 생성한다. child 값 변경은 부모에 영향을 미치지 않는다.
2. vfock() : child 프로세스를 생성한다. child 값 변경은 부모에 영향을 미친다.
3. wait() : 자식 프로세스가 끝날 때까지 대기한다.
4. exec() : 다른 프로그램을 실행한다.

### 인터럽트 vs 폴링
인터럽트 : 프로그램을 실행하는 도중에 예외상황 발생시 현재 작업을 중단하고 발생된 상황을 처리한 후 다시 원래로 복귀하는 것
폴링 : 주기적으로 상태를 검사하여 전송을 제어

### 동기 vs 비동기
동기는 시작이나 결과가 동시에 발생하는 것이며, 비동기는 동시에 발생하지 않는 것을 뜻합니다.

### 블로킹 vs 논블로킹
제어권을 어떻게 처리하는지에 대한 차이입니다.
A함수가 B함수를 호출할 때 B에게 제어권을 넘겨주면 블로킹입니다. 반대로 호출한 함수에 제어권을 넘겨주지 않으면 논블로킹입니다.

### 입출력 처리방식
1. CPU
2. DMA(Direct Memory Access) : CPU 개입 없이 주기억장치에 직접 접근하여 입출력을 행하는 방식
3. 채널 : 지능적이지 못한 DMA 방식의 문제점을 해결

### 메모리란?
메모리는 컴퓨터에서 작업을 수행하기 위해 처리 대상이나 결과 등을 저장하기 위한 공간
메모리구조(RAM)
4가지 영역 : 코드, 데이터, 힙, 스택
- 코드영역 : 소스 코드
- 데이터영역 : 전역, 정적 변수
- 힙 영역 : 사용자가 관리하는 영역 (런 타임에 크기 결정)
- 스택 영역 : 함수의 지역, 매개변수 (컴파일 타임에 크기 결정)

### MMU (Memory Management Unit)
CPU가 메모리에 접근하는 것을 관리합니다.

### 메모리 할당 알고리즘
최초, 최적, 최악 적합

### IPC
프로세스 간 통신

### PCB (Process Control Block
OS가 프로세스 스케줄링을 위해 프로세스에 관한 모든 정보를 가지고 있는 DB
커널 쓰레드/유저 쓰레드
커널 쓰레드 : 개별 관리 가능
유저 쓰레드 : 하나의 쓰레드가 중단되면 프로세스 내 모든 쓰레드가 중단

### 프로세스/쓰레드
프로세스 : 실행중인 프로그램
쓰레드 : 프로세스 안의 실행 단위
- 프로세스는 프로세스마다 메모리와 CPU를 할당받아서 사용하는데, 쓰레드는 다른 쓰레드와 메모리와 CPU를 공유해서 사용
- 프로세스는 여러개의 쓰레드를 가짐. 쓰레드끼리 자원을 공유(코드, 데이터, 힙). but 스택은 공유하지 않음

### 멀티 쓰레드 프로그래밍 작성시 유의점
 동기화에 주의해야 한다. 공유 데이터를 명확히 하고 그 외의 접근을 막아야 한다.
 모든 쓰레드가 공유하는 공간인 코드, 데이터, 힙과 쓰레드마다 별도로 존재하는 스택에 대해 파악해 코드를 작성해야 한다.

### 멀티프로세스 vs 멀티쓰레드
하나의 프로그램을 여러개의 프로세스/쓰레드로 구성각 프로세스가 1개의 작업 처리
- 멀티프로세스 : 1개의 프로세스 죽어도 다른 프로세스 계속실행. 독립적인 메모리 할당으로 통신이 어렵다.
- 멀티쓰레드 : 1개의 쓰레드에 문제가 생기면 전체가 영향받음. 메모리를 공유하기 때문에 통신이 쉽다.

### Thread safe
멀티 스레드 프로그래밍에서 여러 스레드로부터 동시에 접근이 이루어져도 프로그램의 실행에 문제가 없음을 뜻한다.

### 가상메모리  
모든 프로세스에게 메모리를 할당하기에 메모리 크기가 한계가 있어서 사용하는 방법. 프로세스에서 사용하는 부분만 메모리에 올리고 나머지는 디스크에 보관하는 기법

### 데드락
데드락은 프로세스가 자원을 얻지 못해, 다음 작업을 못하는 상태. 데드락은 네가지 조건이 동시에 발생해야 성립이 가능. 상호배제, 점유대기, 비선점, 순환대기 
- 상호배제 : 하나의 자원은 하나의 프로세스만 소유
- 점유대기 : 프로세스 다른 프로세스가 점유하고 있는 자원을 기다림
- 비선점 : 다른 프로세스의 자원을 빼앗을 수 없음
- 순환대기 : 자원을 기다리는 프로세스들이 순환 형태

### Race Condition
 두 개 이상의 쓰레드가 공통 자원의 접근 순서가 결과값에 영향을 줄 수 있는 상태

### 동기화 종류
1. Critical section
2. 뮤텍스
3. 세마포어

### 세마포어 vs 뮤텍스
자원에 대한 접근을 동기화하기 위해 사용되는 상호배제 기술이다. 
 뮤텍스는 Locking 메커니즘으로 락을 걸은 쓰레드만이 임계 영역을 나갈때 락을 해제할 수 있습니다. 하지만 세마포어는 Signaling 메커니즘으로 락을 걸지 않은 쓰레드도 signal을 사용해 락을 해제할 수 있습니다. 세마포어의 카운트를 1로 설정하면 뮤텍스처럼 활용할 수 있습니다.

### Critical Section
공유 자원에 대해 접근하는 코드 영역

### SpinLock
다른 쓰레드가 lock을 소유하고 있다면 그 lock이 반환될 때까지 계속 확인하며 기다리는 것

### CPU 스케줄러
준비큐에 있는 프로세스에 대해서 CPU를 할당하는 방법.
FIFO, SJF, Priority scheduling, SRT, Round Robin
1. 비선점 : 먼저 들어온 순서대로 수행
 - FIFO : 먼저 들어온 순서대로 수행
 - SJF : 짧은 수행시간 순서대로 
 - priority scehdul : 우선순위 스케줄링
     : starvation(기아현상) : 우선순위 높은 계속 작업
     -> 에이징 : 머무는 시간 증가하면 우선순위 높여줌
2. 선점 : 현 프로세스보다 높은 순위의 프로세스가 나타나는 스케줄링
 - SRT : 잔여시간을 우선으로
 - Round Robin : 시분할 시스템으로 설계된 스케줄링
   : time slice 기반으로 스케줄링. : 너무크면 FIFO

### Context Switching
인터럽트를 발생시켜 CPU에서 실행중인 프로세스를 중단하고, 다른 프로세스를 처리하기 위한 과정

### 병목현상
메모리를 계층화하여 병목현상을 해결합니다. 자주 접근하는 데이터의 경우에는 캐시에 저장하여 접근 속도를 향상시킴으로써 부하를 줄일 수 있습니다. (캐시라인 : 묶음으로 저장, 주소 기록)

### 쓰레싱 (공간X)
메모리 영역에 접근하게 될 때, 메모리에 페이지 부재율이 높은 것을 의미합니다. 

### 페이지 교체 알고리즘, LRU
새로운 페이지를 할당하기 위해 현재 할당된 페이지 중 어느 것과 교체할지 결정하는 방법입니다.
 LRU 알고리즘은 가장 오랫동안 사용되지 않은 페이지를 선택하는 방식입니다. FIFO, OPT 등이 있습니다.

### 메모리 단편화(공간O, 사용X)
메모리 공간이 충분하지만 작게 나뉘어져 할당이 불가능한 상태를 말합니다.
내부 단편화는 프로세스 필요 양보다 더 큰 메모리가 할당되어 낭비되는 것을 말하며
외부 단편화는 메모리 중간에 사용하지 않는 메모리가 많아진 것을 뜻합니다.
해결방법 : 페이징, 세그멘테이션, 메모리풀

### 세그멘테이션
가상 메모리를 서로 크기가 다른 논리적 단위인 세그먼트로 분할하고 메모리를 할당하는 기법

### 메모리 풀
미리 메모리를 확보하고, 필요에 따라 확보한 메모리로부터 할당 및 반환하는 기법
